Candidate Tracker Test Instructions

Follow these instructions to test the candidate tracker.

Set up

  1. Clear out the database. Run sh maintenance/empty-db.sh to clear all of the data out of the hiring database. This also creates a dump of all of the data in the database into a file called "create-full.sql". It also creates an initial user, Bob Brown, with administrative priviledges.
  2. Start a browser and point it to the tracker.

Add a new candidate

  1. Verify that you are required to log in
    1. From the home page, logged out, click "Add a candidate"
    2. Verify that you are required to log in. 
    3. Select "Bob Brown" as your login.  If should be the only option.  Click the "Login" button
    4. Verify that you are taken to the "Add a User" page.
    5. Return to the Home Page.
  2. Test add candidate
    1. From the home page, test different options for adding candidates by adding a fake user and then visiting the page for that user and verifying that the data you entered is correct.
      1. All entry fields for the candidate but no uploaded documents or comments.
      2. All entry fields and one uploaded document
      3. All entry fields and two uploaded documents
      4. All entry fields and comment
      5. A duplicate name - should show you partial matches already in the database
      6. An empty name - should not allow you to do this.
